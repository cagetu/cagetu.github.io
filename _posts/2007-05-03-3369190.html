Using DirectX Shaders in 3ds Max

Drawbacks of DirectX shaders
-3ds max에서, 아직 불안정하고, 버그들이 있음. source file 과 viewport 같은 texture color 가 나타나지 않는등의 이상한 문제들이 발생할 수 있음. 3ds max에서 속도 향상과, 이런 버들을 잡아 주길 바랄 뿐...

FX Effect format for 3ds max
Max는 2가지 Effect parser를 가지고 있음. Default Max parser와 DXSAS 0.8
이것들이 마음에 안들면, 알아서 만드시오..

DirectX Effect Shader를 개발할 때, 아래의 코드를 이용해서, load 할 Effect Parser를 선택할 수 있음.

string ParamID = "0x0"; // Load defualt Max parser
or
string ParamID = "0x0001; // Load DXSAS parser

위의 라인은 effect file 의 첫번째 지시어로, 시작할 때 언급되어야 한다. 또한, effect가 컴파일 될 때의 macro가 정의되어 있다.

_3DSMAX_, 3DSMAX, _MAX_ and MAX

이 macro들은 게임 엔진과 3ds max나 다른 그래픽툴(DCC tools)들 사이에 이펙트들을 공유할 때, 매우 손쉽게 해준다.

DirectX 9 Shader material UI의 shader parameter 들을 보여주기 위해서는, semantics annotations 들을 사용해야만 한다. 예를 들어, UI에서, spinner를 만들기 위해, 아래와 같은 코드를 작성할 수 있다.

float OffsetScale <
  string UIType = "FloatSpinner";
  string UIName = "Offset Scale";
  float UIMin = -1.0f;
  float UIMax = 1.0f;
> = {0.1f};

이 예제에서, annotations들은 "UIType", "UIName", "UIMin", "UIMax" 이다.
이것들은 3ds max에서, "Offset Scale" caption을 가진 -1.0f ~ 1.0f 사이의 float 값에 대한 spinner를 만들기 위함이다.


semantic의 사용을 보여주는 다른 예제:

float4 PS(VS_OUT In) : COLOR0
{
   // Pixel Shader code here
}

이 예에서, "COLOR0"이 pixel shader에 의해 반환되는 값을 설명하는 semantic이다.
pixel shader는 색상값 float4를 반환한다. 그럼, 3ds max에서 DirectX Shader의 통합을 알아보자꾸나.

How to integrate DirectX shaders in 3ds max?


Default DirectX 9 Shader material
Direct9 shader material은 artists들이 DirectX effect format으로 작성된 DirectX9 shader를 읽어서 3ds Max viewports에서 오브젝트들에 적용해 볼 수 있다. 이 material의 이점은 3ds MAx SDS 와 3DXI를 가지고 쉽게 익스포트를 할 수 있다는 것.
하지만, 이 Material은 매번 아티스트들이 DirectX9 shader를 생성하고, effect file을 로드하고 봐야만 한다는 것 때문에, 아트스트들에게는 손쉬운 방법이 아니다. 그래서, parser에 의해 생성된 auto ui가 완벽하지는 않지만, 최소한의 노력으로 최대한의 효율을 얻을 수는 있다.(ergonomic)

Scripted materials

DirectX9 Shader Material은 MAXScript로 확장할 수 있다. With scripted materials, the UI generated is customized, be sure to validate data and check for errors.
매터리얼들은 관리와 업데이트가 쉽다. 확장된 DirectX9 Shader scripted Material은 effect parameter들을 캡슐화 할 수 있다. 그래서, 같은 종류의 스크립트를 개발할 때, 각각의 effect parameter에 대해, paramblock안에 parameter를 만드는 것을 기억할 수 있다.

Example of a basic scripted plug-in which extends DirectX 9 Shader material:

FX_DEFAULTPATH = “$maps\\fx\\"
FX_TEXTURE = "Texture.fx"

plugin material MyExtendFX
name:"MyExtendFX"
classID:#( 645812,447325 )
extends:DirectX_9_Shader replaceUI:true version:1
(
  parameters MainParams rollout:RollParams
  (
    DiffuseMap type:#bitmap
  )

  rollout RollParams "Material Parameters"
  (
    button btnDiffuseMap "None"
  )

  on create do
  (
    -- Setup initial material
    delegate.effectfile = DEFAULTPATHFX + FX_TEXTURE
  )
)

DirectX9 Shader를 확장할 때, 우리는 "delegate" 변수를 이용해서 이 매터리얼의 properties들을 얻어오고, 설정하고, 접근할 수 있다.

The delegate variable holds:
 -The properties of the DirectX 9 Shader
 -The parameters defined in the effect

우리는 아래와 같이 effect에서 parameter를 작성할 수 있다.

bool g_DiffuseMapEnable <
  string UIName = "Diffuse Map Enable";
> = false;

그리고, material parameter block안에, 아래와 같이 이 parameter를 사용한다.

-- Set DiffuseMap activation value
on DiffuseMapEnable set val do
  delegate.g_DiffuseMapEnable = val

material parameter block안에, 우리가 color, boolean, integer 등등을 설정할 수 있다. 하지만, bitmaps에 대한 값은 설정할 수 없고, rollout part에서만 bitmap들의 값들을 설정할 수 있다.

To set the bitmaps you can try:

Effect code:

texture g_ Diffuse Texture <
  string UIName = "Diffuse Map";
  int Texcoord = 0;
  int MapChannel = 1;
>;

Scripted material code (in the rollout bloc):

-- When pressing btn DiffuseMap button
on btn DiffuseMap pressed do
(
  bmp = selectbitmap caption: "Diffuse Map"
  if(bmp != undefined) do
  (
    delegate.g_ Diffuse Texture = bmp  -- Set effect bitmap
    btnDiffuseMap.text = bmp.filename  -- Change caption text
    DiffuseMap = bmp                   -- Set Paramblock value
  )
)

Write your own material with the SDK

3ds Max SDK를 이용해서, 우리가 DirectX hardware shader material을 개발할 수 있다.
SDK를 이용해서, DirectX Shader들을 만드는 방법은 2가지가 있음 : Material or DirectX Manager plugin
(In my opinion, I think the Material plug-in is better because it's a true material, contrary to a DirectX Manager plug-in which is part of the Standard material, and the is easier for a graphic artist to use in Max.)

자 이제 어떻게 effec shader들을 export하는지 알아볼자꾸나.


Exporting DirectX shaders

With the SDK

3ds Max SDK는 DirectX 9 Shader materials and scripted materials를 접근할 수 있다.
우리는 bitmap, light, effect properties들을 얻을 수 있다.
To get all DirectX 9 Shader and scripted materials extending DirectX 9 Shader material that are in the scene,
you can use this code:

// Get scene materials
Interface* ip = GetCOREInterface();
MtlBaseLib* lib = ip->GetSceneMtls();
if (!lib)
  return 0;

// Loop over the scene materials
const int NumMtls = lib->Count();
for( int i = 0; i < NumMtls; i++ ) {
  // Get current material
  MtlBase* mtl = static_cast<MtlBase*>( (*lib)[ i ] );
  if (!mtl)
    continue;

  // DxMaterial or extend ?
  IDxMaterial* dxMtl =
  (IDxMaterial*)mtl->GetInterface(IDXMATERIAL_INTERFACE);
  if( dxMtl)
    ProcessDxMaterial(dxMtl);
}

DxMaterial을 처리할 때, 우리는 material properties에 접근하여 사용할 수 있다.

// Get informations from the DirectX Shader material
// Get number of bitmaps used
unsigned int NumberOfBmps = dxMtl -> GetNumberOfEffectBitmaps ();

이 코드는 DirectX9 Shader 매터리얼 or 이 확장된 매터리얼을 가지고 작업한다.

그리고, 우리는 index 0으로 parameter block을 사용하여, effect parameter들을 얻을 수 있다.

// GetParamBlock() work only on the material pointer
IParamBlock2* pblock = mtl ->GetParamBlock(0);

This time, the previous code works only with DirectX 9 Shader.
Running this code on an extend DirectX 9 Shader, the value returned is the scripted material parameter block and not the effect parameters.

With 3DXI

3DXI (known as IGame) is a high level API for 3ds Max SDK. With this API you can easily and quickly extract data from Max. 3DXI is very interesting in order to parse an effect file used in a DirectX Shader. In 3DXI, in order to loop over the scene materials you have to use the root materials. To recognize a DirectX 9 Shader material, the 3DXI material class provides the function GetIGameFX() returning the 3DXI effect representation of this material.

To get all DirectX 9 Shader materials that are in the scene, you can use this code:

// Init IGame
IGameScene* pIgame = GetIGameInterface();
pIgame->InitialiseIGame(false);
pIgame->SetStaticFrame(0);

unsigned int i = 0;

// Loop over scene materials
for(i = 0; i < pIgame->GetRootMaterialCount(); i++)
{
  // Get this root material
  IGameMaterial* mtl = pIgame->GetRootMaterial(i);
  if(!mtl)
    return;

  // DxMaterial ?
  IGameFX* fx = mtl->GetIGameFX();
  if(fx)
    ProcessEffect(fx)
}

To loop over the effect technique and passes you can try this code:

unsigned int i = 0, j = 0;

// Loop over the technique of the effect
for(i = 0; i < fx->GetNumberOfTechniques(); i++)
{
  // Get current technique
  IGameFXTechnique* tech = fx->GetIGameFXTechnique(i);
  ProcessFXTechnique(tech);

  // Loop over the passes of the current technique
  for(j = 0 ; j < tech->GetNumberOfPasses(); j++)
  {
    // Get current pass
    IGameFXPass* pass = tech-> GetIGameFXPass(j);
    ProcessFXPass(pass);
  }
}

3DXI is an interesting library for import/export in constant development.
DirectX shaders support is moving along more and more.

With MAXScript

If you only want to export some basic effect properties and informations,
you can use MAXScript. In script, we can access bitmaps properties and some common informations. To access DirectX 9 Shader material properties you can use the following MAXScript code:

-- Loop over scene materials
for mtl in sceneMaterials do
(
  -- DxMaterial?
  if(Classof mtl == DirectX_9_Shader) do
  (
    -- Get current technique
    CurrentTechnique = mtl.technique
    EffectFile = mtl.effectfile

    -- Get used bitmap(s) in the effect
    for i = 1 to mtl.numberofbitmaps() do
    (
      bmp = mtl.geteffectbitmap i
      BmpFileName = bmp.filename
    )
  )
)

This small script shows how to access current effect filename, the index of the technique used and the bitmaps' filenames.

Conclusion

In conclusion, DirectX shaders can be integrated easily in 3ds Max. The development of a DirectX shaders effect for Max isn't difficult. We can integrate Direct shaders as an entire material with scripted materials or with the SDK and export them using the SDK and 3DXI DirectX shaders support in 3ds Max is extended more and more. So, if you are not yet using DirectX shaders in Max, I think it's time to think about it.
