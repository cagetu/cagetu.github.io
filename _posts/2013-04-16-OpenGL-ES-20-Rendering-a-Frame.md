<a href="https://developer.apple.com/library/ios/#documentation/3DDrawing/Conceptual/OpenGLES_ProgrammingGuide/WorkingwithEAGLContexts/WorkingwithEAGLContexts.html#//apple_ref/doc/uid/TP40008793-CH103-SW12" target="_blank">OpenGL ES Programming Guide for iOS &gt; Rendering a Frame</a>에 있는 내용을 정리!!<div><br/><div><img alt="" loading="lazy" src="/assets/images/posts/5737164/c0001532_516cacf7e9358.jpg"/></div><div><br/><div><br/></div></div><div><b>* Erase Renderbuffers</b></div><div>- 다음 프레임을 그리기 위해 필요하지 않는 이전 프레임의 모든 renderbuffer들의 내용들을 지운다. glClear 를 이용</div><div>- 수동적으로 buffer를 지우는 것보다 glClear를 사용하는 것이 더 효율적일 뿐 아니라, <u>glClear 사용은 존재하는 내용들이 discard 될 수 있다고 OpenGL ES에게 알린다(hint)</u>. 몇몇 그래픽 하드웨어에서, 이는 메모리로 이전 내용들을 로드하는 메모리 연산을 피하도록 한다.</div><div><br/></div><div><b>* Prepare Resources</b></div><div>- frame에 렌더링하는데 필요한 모든 OpenGL ES objects를 준비한다. (vertex buffer objects, textures 등등... )</div><div>(역주: 실제로 대역폭이 낮은 하드웨어의 경우, 여기서 엄청난 병목이 발생하기도 한다. 특히, 이전 화면사이즈 렌더타겟을 가지고 올 경우에는 지연 현상이 발생한다.)</div><div><br/></div><div><b>* Execute Drawing Commands</b></div><div>- 이전 스텝에서 준비한 objects들을 이용하여 drawing commands를 submit 한다. </div><div>- 효율적으로 동작하는 rendering code를 구현하기 위해서는 <a href="https://developer.apple.com/library/ios/#documentation/3DDrawing/Conceptual/OpenGLES_ProgrammingGuide/OpenGLESApplicationDesign/OpenGLESApplicationDesign.html#//apple_ref/doc/uid/TP40008793-CH6-SW1" target="_blank">"OpenGL ES Application Design Guildelines"</a>를 참고하자.</div><div><br/></div><div><b>* Reslove multisampling</b></div><div>- 멀티샘플링을 사용하다면, 어플리케이션은 <u>사용자가 present하기 전에 </u><u>pixel을 resolve해야만 한다</u>. </div><div>- <a href="http://cagetu.egloos.com/5737136" target="_blank">OpenGL ES 2.0 MSAA 사용법</a> 참고</div><div><br/></div><div>* <b>Discard unneeded Renderbuffers</b></div><div>- discard 연산은 iOS 4.0 이상에서 사용 가능 (EXT_discard_framebuffer)</div><div>- discard는 OpenGL ES에 대한 performance hint이다. discard command가 완료된 후에 어플리케이션에 의해 (하나 이상의) renderbuffer들의 내용이 사용되지 않는다고 OpenGL ES에게 말한다. </div><div>- 어플리케이션은 renderbuffer의 내용들을 필요하지 않는다고 OpenGL ES에 힌트를 알려줌으로, 버퍼안에 데이터는 discard되거나 버퍼의 내용을 업데이트하기 위해 유지하는 비싼 작업들을 피할 수 있게 된다.</div><div>- 어플리케이션이 frame에 대해서 모든 drawing command를 제출(submit)한 상태에서, 화면에 출력하기 위해서 color renderbuffer는 필요할 것이지만, 아마도 depth buffer의 내용들은 필요가 없을 것이다. 이럴 때 discard를 이용하여 depth buffer의 내용들을 discard한다.</div><div><br/></div><div><b>*Present Results</b></div><div>- 기본적으로, 우리는 renderbuffer의 내용들은 어플리케이션이 renderbuffer를 present한 후에 discard된다고 가정해야만 한다.</div><div>- 이것은 매번 어플리케이션은 frame을 present해야한다는 것을 의미하고, 새로운 frame을 렌더할 때 frame의 내용들을 완전히 재생성 되어만 한다. (color buffer를 erase 하는 이유가 바로 이것이다)</div><div>- 어플리케이션이 frame 간의 color buffer의 내용을 보전하고 싶다면, CAEAGLLayer에서 kEAGLDrawablePropertyRetainedBacking 키를 추가하고, glClear에서 GL_COLOR_BUFFER_BIT"를 사용하지 않는다. <span style="font-family: Verdana; font-size: small;">(역주 : iOS말고도 이 기능이 사용가능한지 궁금하기는 하네?! 되도록 사용하지 않는 것으로.. ㅎㅎ)</span></div><div>- 역주 : 픽셀 연산(알파 블랜딩 등)을 많이 사용하게 되면, 여기서 엄청 병목이 발생~ </div><div><br></div><div>OpenGL ES 에 대해서는 은근히 책을 봐도 딱 알맞는 자료가 없는데, 그나마 Apple 사이트가 갑인 듯!!!</div><div><span style="font-family: Verdana; font-size: 9pt;">알아두면 피가 되고 살이되는 소중한 내용들.... ㅎㅎㅎ</span></div></div><div><br/></div>
