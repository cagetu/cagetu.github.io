--- layout: post title: "[Vulkan 일지] 머터리얼 및 셰이더 파라미터 바인딩 재구성 (2)" date: 2017-04-25 01:15:46 +0900 categories: [개발] --- [Vulkan 일지] 머터리얼 및 셰이더 파라미터 바인딩 재구성해야지!! 라고 생각하다가 살짝 정신이 돌아온 틈을 타서 좀 해봤다.

OpenGL ES에서 셰이더 처리는 Vertex.vert, Frag.frag 라는 읽어서 하나의 셰이더에 해당하는 Program이라는 녀석에서 Attach 시켜주면, 그 다음에는 이 Program에 연결된 모든 셰이더들에 대한 파라미터 정보를 가지고 와서 내부적으로 Binding해서 사용했었다.

       // 셰이더 링크 (이제 조회가 가능하다.)
       glLinkProgram(m_Program);
       GLint Linked;
       glGetProgramiv(m_Program, GL_LINK_STATUS, &Linked);
       if (!Linked)
       {
              int i32InfoLogLength, i32CharsWritten;
              glGetProgramiv(m_Program, GL_INFO_LOG_LENGTH, &i32InfoLogLength);
              char* pszInfoLog = new char[i32InfoLogLength];
              glGetProgramInfoLog(m_Program, i32InfoLogLength, &i32CharsWritten, pszInfoLog);
              PlatformUtil::DebugOutFormat("Shader Link Fail: %s\n", pszInfoLog);
              delete[] pszInfoLog;
              return false;
       }

하지만, Vulkan에서는 VertexShader, FragmentShader는 OpenGLES 처럼 하나의 Shader에서 파라미터를 모아서 처리하는 형태가 아니기 때문에, 각각 셰이더 파일에 대응하는 셰이더 객체를 만들고 UniformBuffer를 할당해서 Binding하는 식이 필요했다. 

그래서, Shader는 ShaderBindParam을 가지도록 하고, ShaderBindParam(VERTEXSHADER | PIXELSHADER) 형태로 셰이더 바인더의 타입을 설정할 수 있게 해주어서, OpenGL ES에서는 VERTEX와 PIXELSHADER가 모두 바인딩하고, Vulkan에서는 VERTEX, PIXEL을 별도로 만들어서 붙여주었다. 일단 공통된 형태로 처리가 가능해지기는 했다. 그게 대단한 것은 아니지만, 정리하는 차원에서.. ;;;
