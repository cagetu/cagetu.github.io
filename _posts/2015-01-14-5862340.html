--- layout: post title: "[VisualShock] 멀티플랫폼 코드 관리" date: 2015-01-14 02:10:22 +0900 --- 엔진 디렉토리 구성을 얼추 마치고 나서 본격적으로 엔진 코드를 작성해보려고 하는데, 어떻게 플랫폼 별 코드를 작성하고 관리하는 것이 좋을지에 대해서 고민에 빠졌다.

이전에 사용하던 방식은 하나의 클래스를 선언하고 내부에서 기능을 플랫폼 별로 #if ~ #endif 로 처리하는 방식이었다. 예를 들면,

bool OutputConsole::Show(bool show)
{
#if defined (WINDOWS)
// 윈도우용 처리
#elif defined (__APPLE__)
// iOS용 처리
#elif defined (ANDROID)
// 안드로이드 용 처리
#endif
}

이 방식이 솔직히 만드는 입장에서 빠르게 만들 수 있어서 좋기는 한데, 확장성이 떨어지고, 많아질수록 관리가 어려워지는 단점이 있다. 엔진 리서치를 해보니까, (물론 이런 일은 없겠지만) 지원해야 하는 플랫폼이 무지 다양하고, 현재 몇 개만 지원한다고 하더라도 다양한 플랫폼으로 확장할 수 있도록 구현되어야 한다는 것을 알 수 있었다.

그래서 어떻게 하면 멀티플랫폼 용으로 코드를 분할해서 만들 수 있을까? 언리얼4 코드를 보면 대략 다음과 같이 되어 있다.

* Platform.h

...

#include "GenericPlatform/GenericPlatform.h"

#if PLATFORM_WINDOWS
#include "Windows/WindowPlatform.h"
#elif PLATFORM_ANDROID
#include "Android/AndroidPlatform.h"
#elif PLATFORM_IOS
#include "IOS/IOSPlatform.h"
#endif

즉, 기본형을 만들어 놓고, 해당 플랫폼 별로 구현체를 만들어 놓은 다음에 해당 플랫폼 코드만 포함시도록 되어 있더라. 특이한 것은 이전에 이런 것들은 복잡한 상속을 통해서만 처리했던 것 같은데, 상속을 간결하게 사용하도록 유지되어 있다는 것이다. 사실 이것이 난이도가 더 높아 보인다. ㅡㅡ

다른 (오픈 소스) 엔진을 보아도 대부분 이런 형식으로 구현 내용은 전부 집어 넣어두고, 포함 관계를 선택하는 방식인 것 같다.
