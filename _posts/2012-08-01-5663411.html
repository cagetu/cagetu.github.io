포스트프로세싱을 처리할 때, Screen Aligned Screen Quad를 이용하여 렌더링을 할 때, 깊이 버퍼를 이용해서 포지션으로 변환을 하는 경우, Depth To Position을 참고하면 된다.

하지만, 이미지 공간에서 샘플링을 해서 샘플링한 픽셀의 Depth를 Position으로 변환을 할 때에 저 방식을 이용하면 문제가 생길 수 있다. (이 캐삽질.. ㅡㅡ;;)

그 이유는 버텍스 셰이더에서 현재 위치에 Pixel에 맞는 view ray를 얻어와서 사용하는 방식인데, 만약 옆 픽셀을 샘플링한 depth을 포지션을 변환하면, 현재 픽셀의 view ray를 가지고 사용하기 때문에 오차가 발생한다. 이를 해결하기 위해서는 view ray도 한 픽셀 크기 만큼 옆으로 같이 이동시켜주어야 한다.

좀 더 쉽게 처리하기 위해서는 걍 projection 정보를 역으로 산출하면 편한데, 
http://jcoluna.wordpress.com/2011/01/27/reconstructing-view-space-position-from-depth/를 참고하면 설명이 잘 되어 있다.

그냥 NVIDIA 샘플코드에 나와 있는 방식으로 쉽게 쉽게 처리하면, 
(NVIDIA SDK10 "Screen Space Ambient Occlusion")

float3 fetch_eye_pos(float2 uv)
{
        float eye_z = tex2D(DepthBuffer, uv).x; // view 공간 깊이값

        uv = (uv * float2( 2.0, -2.0 ) - float2(1.0, - 1.0));
       
        const float cotan_halffov = CameraFov.y;
        const float aspectratio = CameraFov.z;
        const float2 focallen = float2(cotan_halffov * aspectratio, cotan_halffov);

       return float3(uv / focallen * eye_z, eye_z);
}

이렇게 사용하면, 이미지 공간에서 이동한 uv에 대해서도 view space 포지션을 구할 수 있당....
