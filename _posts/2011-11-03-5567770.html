--- layout: post title: "HLSL - half vs float" date: 2011-11-03 12:15:35 +0900 categories: [개발] --- 스칼라 타입
 타입 값
 bool true 또는 false 를 갖으며, C의 bool 과 같다.
 int 32비트 부호있는 정수, C의 int 와 같다.
 half 16비트 부동소수점, float 의 반토막
 float 32비트 부동소수점, C의 float 과 같다.
 double 64비트 부동소수점, C의 double과 같다.


셰이더를 작성하면서, 값의 범위에 따라서, 변수 타입을 적절하게 선택해서 사용하는 것이 최적화에 도움이 된다는 이야기는 많이 들었지만, 거의 float 만 사용하는데, 버릇(?)을 고쳐보려고, 가볍게 정리!

half의 경우에는 color 기반이나, normalize 된 결과에는 정밀도 손실없이 충분히 사용이 가능함. 
아직은 많이 써보지 않아서, 어디에 써야할지 잘 모르겠지만, 다른 작업 소스를 참고하면서 습관해 해보자.... 
(이 습관을 잘 들여두면, OpenGLES로 작업을 할 때에도, 분명 도움이 될 것임....)

DX10+ 부터는 half를 써도, 간단한 계산의 경우에는 별 차이가 없을 것이라고 하는데, 시대가 좋아져서, 결과가 어떻게 될지는 잘 모르겠네요... NVIDIA나 ATI 의 최신 문서들을 보면, 최적화 문서들이 매번 업데이트 되니까, 주기적으로 봐두면 좋겠죠?! 하지만, 최근에는 DX11 용 내용이 많아서, 주로 DX9 기반으로 작업을 하는 국내자들은 이전 문서에서 더 볼 내용이 많은 듯.
(Geforce7 시리즈 문서 : http://developer.download.nvidia.com/GPU_Programming_Guide/GPU_Programming_Guide.pdf)

다른 기본적은 최적화 기법은 DirectX9에서의 현대 GPU 최적화 여기를 참고하면 되겠습니다. 
정리가 잘 되어 있네요... (님 짱이심!!!)
