--- layout: post title: "[VisualShock] 멀티플랫폼 + 그래픽 api 코드 관리" date: 2015-01-21 08:24:51 +0900 categories: [개발] --- 코드를 작성해 보면서, 여전히 멀티 플랫폼 코드 분리에 대해서 고민 중이다. 일단 이 부분은 언리얼4가 제일 정리가 잘 되 있는 것 같아서 참고 (라고 읽고 따라...) 하면서 해보고 있다. 

일단 기본적으로 플랫폼 분류를 해서 진행하고 있는데 또 복잡한 것이 발생했다... 바로 그래픽 API 코드 분리이다. 일단 시작은 OpenGL ES 2.0 / 3.0 으로 하려고 하지만, 확장성을 고려해서 PC 쪽에서 사용할 수 있는 DirectX11 이나 OpenGL을 붙일 수도 있고, glNext나 DirectX12를 붙일 수도 있다. 아무튼 확장성을 고려해야 한다는 것이 중요하다고 판단했다. (사실 이게 지금 고민거리인지 모르겠다. 차라리 OpenGL을 사용하는 전용 엔진을 만들고, 나중에 다시 만드는 것이 현명할지도 모른다. 암튼 이번에는 멀티플랫폼 구성도 공부 목표 중에 하나니까...)

폴더 구성으로 보면서, 이런 형태로 일단 코드를 분리해서 관리하면 될 것 같은데, OpenGL / OpenGL ES 까지 세분화할 필요가 있나 싶기도 하다. 아직 이것도 이리 저리 바꿔보면서 진행 중이라 고민이 많다..

빨리 렌더링 엔진 쪽 작업을 시작하고 싶지만 (사실 이거 하고 싶어서 시작 한거 아님?!), 실험하고 싶은 것이니 이번에 한번 해보고 데이터를 얻어보자. 하지만, 기반 설정이 더뎌서 속도가 안나니 좀 신나지 않는 것은 사실...
