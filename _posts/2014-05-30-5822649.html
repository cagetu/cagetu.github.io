요즘 Hot한 라이브러리 중에 모바일 게임 동영상을 레코딩해서 공유해주는 라이브러리인데, kamcord라는 서비스가 있습니다. 요즘 모바일 게임을 보면 마케팅 경쟁이 정말 치열한데, 대기업이야 자금을 기반으로 한 이벤트와 마케팅을 한다고 하지만, 상대적으로 규모가 작은 업체는 사실 여러가지로 한계가 있다고 생각합니다.

그러다가 이 라이브러리를 보면서 아주 조금은 바이럴을 일으키는데 도움이 될 수 있는 방법이 되지 않을까? 라는 호기심 반 기대 반으로 한번 테스트해보고 싶다는 생각이 들었습니다. (그래서, 지난 주부터 사실 이거 해보려고 안드로이드 빌드를 열심히 한 거임!!)

일단 국내에서는 Android를 많이 사용하기 때문에, Android를 중심으로 시작을 해봅니다. kamcord - android 연동 방법을 보시면 아시겠지만, 설정하는 것 자체는 매우 쉽네요. 
1. SDK의 AndroidManifest.xml에 있는 내용을 대상 프로젝트의 AnroidManifest.xml에 넣어주고
2. Library와 Code를 대상 프로젝트에 넣어주고, 이클립스 프로젝트에서 임포트 시켜준다.
3. Kamcord 라이브러리들을 볼 수 있도록 ndk 빌드 프로세스에 연결해준다. (jni/Android.mk에 추가)
4. SDK에 있는 res 를 대상 프로젝트에 추가해준다.

그런데 빌드하다가 보니까, 다음과 같은 에러가 발생했습니다.

찾아보니까, TextView, fontFamily는 Android API Level 16 이후에 추가된 것이라고 하니, Andorid API 17 이상을 사용해야 합니다. 즉, 4.2.2면 Jelly Bean인데 그 정도면 그 밑의 OS 버전은 무시하고 가면 되지 않을까 싶네요.
컴파일을 해보니, 일단 빌드가 되네요.. 자~ 이제 설정은 끝났네요. 

그럼 이제 Kamcord API를 어떻게 사용하는지 보면 될 듯 하네요. Java API를 통해서 사용하는 방법이 있고, C++ API를 통해서 호출하는 방법이 있는 듯 합니다. 


일단 보기에는 간단하게 Java API만 사용해서 되는 것 같기는 한데 아직 이 부분을 테스트를 진행해보지 않아서, 실제로 한번 실행해보고 리뷰해보도록 하겠습니다.
