--- layout: post title: "[VisualShock] 유니티 멀티 카메라" date: 2015-05-08 03:22:30 +0900 --- 유니티 엔진을 배워보려고 책을 일단 보고 있지만, 아직 어떻게 사용하는지는 잘 모른다. (뭐라도 빨리 만들어봐야.. ㅡㅡ;)

스터디모임에 가서 "멀티 카메라" 사용에 대해서 이상한 부분이 있다고 이야기를 나누었는데, 구동 방식이 매우 흥미로웠다.

먼저, 유니티 책을 보면서 느낀 점이기는 한데, Orge3D 엔진과 구성과 작동 방식, 사용하는 용어가 굉장히 비슷한 부분이 많다.
(개인적인 추측으로는 Orge3D 엔진을 많이 다루어 본 사람이 참여했을 것이라고 본다.) 

멀티 카메라를 사용하는 방식을 보니까, 메인 카메라로 장면을 렌더링하고, UI 카메라로 UI를 렌더링하는데, Clear Color를 하지 않는 형태로 덮어서 사용하는 방식이었다. 이건 마치 RenderPass를 카메라로 사용하는 것으로 보이는데, 카메라로 매핑하니까 굉장히 직관적이고 사용하기 편해보였다. 거기에다가 Camera가 장면 그래프에 Attach될수도 있기 때문에 특정 오브젝트에 위치를 계승할수도 있으니, 다양한 사용이 가능해졌다.

Unity Tutorial - Multi Camera FPS Setup 


지금은 RenderPass를 연결해서 하나의 프레임을 구성하는 형태를 사용하는데, 이 부분은 예전에 CindyEngine에서 unity3D와 비슷하게 구성한 기억이 있으니, 적절히 배합해서 유니티의 장점(유연함, 직관성)을 좀 가져와야겠다.

* 유니티를 좀 더 써봐야겠다. 재미있는 것들이 많네..
* 역시 SceneGraph 방식이 유연하기는 한 듯.


2015.05.11
유니티의 경우 카메라가 하나의 렌더 패스 (렌더 타겟)으로 사용된다. 하나의 카메라는 하나의 뷰포트를 가지고, 하나의 렌더 타겟을 가지는 듯 하다. 여기서 가지는 렌더 타겟은 모든 처리가 끝난 렌더 타겟 텍스쳐 인 듯.

현재 VisualShock의 경우, 렌더 패스가 카메라와 독립적인 단위이기 때문에 이렇게 처리해서 멀티카메라를 가능하도록 한다. 사실 카메라가 렌더패스라는 개념보다 훨씬 직관적이다(내가 졌다. ㅠㅠ). 결국 이건 툴에서 어떻게 접근할 수 있게 하는지에 대해서 표현을 잘해서 해결해야 할 듯..

마지막으로 카메라 관련한 처리가 필요한데, 지금까지 카메라가 장면의 오브젝트에 Attach 되는 상황 (씬 그래프 구조)를 생각하고 있지 않았기 때문에, 고민을 좀 해봐야겠다.
