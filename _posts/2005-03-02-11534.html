참조 카운트란?
어떠한 객체를 참조하고 있는 횟수를 기록하는 것. 즉, 객체의 생명주기를 관리하는 하나의 방법이라 말할 수 있다.
이 횟수가 있다는 증거는 어디에 선가 쓰이고 있다는 것을 의미하므로, 함부로 지웠다가는 어디에선가 사용하고 있는 정보가 무효화 된다는 것이다. 따라서 이 횟수가 0이 되기전에는 이 객체가 삭제되지 않는다.
횟수가 0이 된다는 것은 그 누구도 참조하지 않는다는 것으로 그 객체가 삭제되므로서 잘못된 메모리참조를 범하지 않게하는 기술이다.

<예제>
class cReferenceCounter
{
public:
    intnRef;
    int AddRef()
    {
      return ++nRef;
    }

    voidRelease()
    {
      if( --nRef <= 0 )
      {
        delete this;
      }
    }

};

간단한 예제이지만, 대충 참조카운트란 저런 것이다. 이와 같이 참조 카운트에 대한 클래스를 만들어 두고 필요할 때 상속을 받아서 객체에 대해 참조카운트를 높이고, 사용을 마쳤을 때, Release()를 호출하면 된다.

<주의할 점>
객체가 삭제되는 시점은 delete this이다. 이것은 스스로 삭제하는 것으로 DLL내보내기로 만들어진 객체나 그것을 상속받은 외부 class등 복잡한 형식에서는 문제가 발생할 요지가 있다. 즉, 서로다른 메모리영역을 사용하는곳에서 생성하였을 경우는 삭제도 같은 곳에서 이루어지기 위해서 delete this부분을 다시한번 재선언해줄 필요가 있다.
