--- layout: post title: "유니티 Mobile Shader" date: 2016-01-21 02:43:41 +0900 categories: [개발] --- 유니티를 사용하게 되었는데, 전혀 유니티에 대한 사전 지식도 데이터도 없어서 한 땀 한 땀 익히고 있는 중이다. 

요즘에는 유니티의 라이팅 셰이더가 어떻게 되어 있는지를 좀 살펴보고 있는 중에 애초에 PBS가 어울리지 않는 프로젝트이기도 하고 솔직히 Standard 셰이더는 좀 별로인 것 같아서, 그냥 셰이더를 처음부터 직접 하나 하나 구현해보면서 역으로 유니티에서 라이팅 계산을 어떻게 하는지를 구현해보고 있다.

특히 눈에 띄는 것이 Mobile 폴더에 있는 내용인데, 셰이더 파일을 보면 되게 단순해보인다. suf 부분만 작성해주면 나머지는 자동으로 구현을 추가해주는 식인 것 같다. 

먼저 Mobile/Diffuse를 적용해보았다. 그냥 생각하기에 Lambert 라이팅에 텍스쳐 칼라를 가져와서 곱해주는 것 같아 보인다. 

그래서 구현해 봤다.

surface 셰이더라는 것이 내부적으로 뭔가를 더 해주는 것 같다. 기본적으로 봤을 때, Lambert만 해서 음영지는 부분이 저렇게 밝을 수가 없다. 뭔가 ambient 부분을 추가해주는 코드가 있는 것 같았다. 그러다 Aras님의 블로그에서 완전체의 구현으로 추정되는 코드를 발견(http://aras-p.info/blog/2010/07/16/surface-shaders-one-year-later/) 역시 내부자!!!

SH를 이용한 환경광을 추가해주는 듯 하다. 만들어보자!

맞췄다!!! 코드로 설명하면 대충 이런 느낌이다.


v2f vert(appdata v)
{
       v2f o;
       o.vertex = mul(UNITY_MATRIX_MVP, v.vertex);
       o.uv = TRANSFORM_TEX(v.uv, _MainTex);

       o.worldPos = mul(_Object2World, v.vertex);
       o.worldNormal = UnityObjectToWorldNormal(v.normal);
       o.worldViewDir = normalize(WorldSpaceViewDir(v.vertex));
       o.color.rgb = ShadeSH9(half4(o.worldNormal, 1.0));

        return o;
}

fixed4 frag(v2f i) : SV_Target
{
       fixed4 result = 0;

        // Surface 셰이딩 구간
       fixed4 albedo = tex2D(_MainTex, i.uv);

       float3 worldViewDir = normalize(i.worldViewDir);
       float3 worldLightDir = normalize(UnityWorldSpaceLightDir(i.worldPos.xyz));
       float3 worldNormal = normalize(i.worldNormal);

        float lambertTerm = max( 0, dot(worldNormal, worldLightDir));

       result.rgb = albedo.rgb * _LightColor0.rgb * lambertTerm;

        // SH 라이팅 추가 (surface 셰이더 후처리)
       result.rgb += i.color * albedo.rgb;

        return result;
}

대충 surface shader 내부가 어떤 느낌인지 약간 감이 온다. 이제는 좀 편하게 이건 저것 뒤져볼 수 있을 듯.. ㅎㅎ
