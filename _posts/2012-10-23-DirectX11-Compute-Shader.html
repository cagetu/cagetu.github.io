*DirectCompute Memory Model

- Register-Based Memory 
: 오직 현재 실행되는 쓰레드로 접근가능하다. 매우 빠르다. 임시 레지스터(temporary register : r#, x#)을 사용한다. 하지만, 제한된 레지스터 개수를 확인해야 한다. (SM 5.0은 4096개)

- Device Memory
: 훨씬 더 넓은 저장 영역이 필요하다. device memory resource로서 shader resource view와 unordered access view를 사용한다. 이 두 리소스 뷰는 읽거나 읽고/쓰기 접근이 가능하다. Register-Based Memory보다 더 느리다. 모든 쓰레드는 하나의 unordered access view를 통해 읽고 쓸 수 있기 때문에, 반드시 동기화 처리가 필요하다.

- Group Shared Memory
하나의 Thread Group 안에 Thread 들은 같은 메모리 영역을 접근하는 것이 가능하다. 이 공유 메모리가 "Group Shared Memory(GSM)"이다 (SM 5.0 : 32k로 제한, SM 4.0 : 16k로 제한). 하나의 쓰레드 그룹 내에서는 모든 쓰레드가 공유할 수 있기 때문에 동기화 처리가 필요하다. 만약 알고리즘이 많은 수의 쓰레드가 공유 메모리 풀에 접근한다면, group shared memory는 접합하지 않다. 

Register based memory는 가장 access가 가장 빠르지만, 가장 작은 양의 메모리 이용이 가능하다. device memory resource는 대용량의 메모리를 제공하지만, 가장 느린 access 속도이다. group shared memory는 앞의 두 개의 중간 정도인데, resource memory보다는 접근 속도가 빠르고, register-based memory보다는 더큰 메모리를 사용할 수 있다. 

Thread Group내에서의 Thread들에 대해서는 GSM을 이용하면, 데이터를 효율적으로 사용할 수 있다.
