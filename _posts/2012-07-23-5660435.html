--- layout: post title: "DirectX11 Map( Lock) 관련 옵션..." date: 2012-07-23 15:33:52 +0900 categories: [개발] --- DirectX9에서는 Lock 이었지만, 11에서는 Map ~ Unmap으로 변경되었다구...

이 녀석의 옵션이 Buffer 생성 옵션과 밀접하고, DirectX9보다 더 설정하는 부분이 세밀해졌음...
DirectX9에서는 그냥 대충해도 적당히 넘어갔었는데, DirectX11에서는 더 타이트해졌음...

Map에서 D3D11_MAP_READ로 값을 읽어오려면, D3D11_CPU_ACCESS_READ로 생성되어야 하고, 이는 D3D11_USAGE_STAGING로 생성할 때만 사용가능...
(D3D11_USAGE_STAGING 는 Pipeline에 Bine 될 수 없다네... 아직 어디에 쓰이는지 정확히 모르겠음..)

    desc.Usage            = D3D11_USAGE_STAGING;
    desc.BindFlags        = NULL;
    desc.CPUAccessFlags   = D3D11_CPU_ACCESS_READ;
    desc.MiscFlags        = 0;

Map에서 D3D11_MAP_WRITE로 값을 쓰려면, D3D11_CPU_ACCESS_WRITE로 생성되어 있어야 하고, 이는 D3D11_USAGE_DYNAMIC or D3D11_USAGE_STAGING 으로 생성되어야 한다.
(Map 함수를 통해서 포인터를 얻어와보면, 쓰기 전용이라 그런지, 값이 없다... ㅡㅡ;;)

D3D11_USAGE_DEFAULT : GPU에 의해서 읽고 쓰기 접근이 가능.
D3D11_USAGE_IMMUTABLE : GPU에서 읽기만 가능. CPU에서 접근 급지! (한번 생성되고 변하지 않는다면. 이렇게...)
D3D11_USAGE_DYNAMIC : GPU에서 읽기만, CPU에서는 쓰기만... Map method를 사용
D3D11_USAGE_STAGING : GPU에서 CPU로 값을 전달.

DirectX9 입장에서 봤을 때에는, UpdateSubresource나 Map ~ Unmap이 전부 Lock ~ Unlock 으로 처리하기 때문에, 아직은 UpdateSubresource와 Map ~ Unmap를 어떤 때에 써야 하는지 정확하게 구분이 되지 않음... 

UpdateSubresource에 대해서도 좀 정리할 필요가 있겠음...
